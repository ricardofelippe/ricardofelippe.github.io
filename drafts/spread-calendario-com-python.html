Calendar Spread¶

Das estratégias acima o Calendar Spread (ou Spread Calendário) é particularmente interessante por explorar opções em diferentes vencimentos.

Em $t = T_0$, onde $T_0$ é o instante inicial no qual a operação é realizada, é necessário as seguintes operações para montar esta estratégia.

  • Venda de uma opção de compra $C_1$ com preço de exercício $K$ e vencimento $T_1$
  • Compra de uma opção de compra $C_2$ com preço de exercício $K$ e vencimento $T_2$

onde $T_1 < T_2$. O custo dessa operação é $C_1-C_2$, ou seja, se embolsa o prêmio da opção de compra $C_1$ e paga-se $C_2$ pela compra da segunda opção. Logo, se $C_1-C_2>0$ esta estratégia é viável para ser executada da forma que foi proposta, já no caso contrário onde $C_1-C_2 < 0$ a estratégia seria viável na posição contrária onde se compra $C_1$ e se vende $C_2$. Aqui eu avalio viabilidade pelo custo da operação sendo positivo, onde isso implica que quem realizar essa operação não teria custo, pois a própria estratégia se paga na largada, não sendo necessário que o investidor pagasse qualquer quantia para montá-la.

Em $t = T_1$, no vencimento da primeira opção temos os seguintes cenários para o ativo objeto $S$:

  • se $S_1 < K$ então $C_1 = 0 \longrightarrow -C_2$
  • se $S_1 >= K$ então $C_1 > 0 \longrightarrow S_{T_1} - K - C_2$

No primeiro cenário com certeza temos uma carteira positiva, dado que $C_2 > 0$, no entanto, no segundo cenário, onde $C_1$ é exercída, não podemos afirmar se a carteira resultante $-S_{T_1} + K + C_2$ é positiva.

Pelo apreçamento neutro ao risco temos:

$$ C_2 = E^Q \left[\max(S_2 - K, 0)\,|\, F_{T_1} \right] $$

e aplicando o lema de Jensen a equação acima temos:

$$ \begin{split} E^Q \left[ \max(S_2 - K, 0)\,|\, F_{T_1} \right] &\ge \max(E^Q[S_2\,|\, F_{T_1}] - K, 0) \\ & = \max(S_1 - K, 0) \end{split} $$

Para o cenário em que $S_1 > K$ temos que:

$$ C_2 = E^Q \left[ \max(S_2 - K, 0) \,|\, F_{T_1} \right] \ge S_1 - K $$

Portanto

$$ -S_1 + K + C_2>0 $$

from numpy import sqrt
from scipy import exp, log
from scipy.stats import norm

def d1d2(S, K, T, r, v, q):
	d1 = ( log(S/K) + (r - q + 0.5*v**2)*T ) / (v*sqrt(T))
	d2 = d1 - v*sqrt(T)
	return d1, d2


def put(S, K, T, r, v, q):
	d1, d2 = d1d2(S, K, T, r, v, q)
	N1 = norm.cdf(d1)
	N2 = norm.cdf(d2)
	
	return -S*exp(-q*T)*(1-N1) + K*exp(-r*T)*(1-N2)


def call(S, K, T, r, v, q):
	d1, d2 = d1d2(S, K, T, r, v, q)
	N1 = norm.cdf(d1)
	N2 = norm.cdf(d2)
	
	return S*exp(-q*T)*N1 - K*exp(-r*T)*N2

def delta(S, K, T, r, v, q):
    d1, d2 = d1d2(S, K, T, r, v, q)
    return exp(-q*T)*norm.cdf(d1)
