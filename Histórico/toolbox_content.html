Command Line Interface

CLI can help:
> Navigate folders
> Create files, folders and programs
> Edit files, folders and programs
> Run computer programs

Special directories: root and home

  • The root directory contains all other directories

  • The root directory is represented by a slash: /

  • Your home directory is represented by a tilde: ~

  • Usually is the name you use to log into your computer

    • CLI Basics 


    > pwd: will display the path to working directory.
> ls: lists files and folders in the current directory
> ls -a: lists hidden and unhidden files  and folders
> cd: stands for "change directory"
> cd .. : allows you to change directory to one level above your current directory
> mkdir: make a directory
> touch : creates an empty file

Ricardo@Usuário-PC MINGW32 ~

2- Entro na pasta criada:
Ricardo@Usuário-PC MINGW32 ~
$ cd LocalRepo
 
3- Realiza-se o link entre Git e GitHub:
Ricardo@Usuário-PC MINGW32 ~/LocalRepo
$ git init
Initialized empty Git repository in C:/Users/Ricardo/LocalRepo/.git/

Ricardo@Usuário-PC MINGW32 ~/LocalRepo (master)
$ git remote add origin https://github.com/ricardofelippe/R.git

Ricardo@Usuário-PC MINGW32 ~/R (master)
$ git clone https://github.com/ricardofelippe/R.git

    Adding 

    If you add new files to a local repository, you need to let Git Know that they need to be tracked

- git add .  : adds all new files
- git add -u : updates tracking for files that changed names or were deleted
- git add -A : does both of the previous
This should do this before committing

    Committing 

    You have changes you want to commit to be saved as an intermediate version
You type the command
- git commit -m "message"

    Pushing 

    You have saved local commits you would like to update on the remote(Github)
Type the command: git push

    Branches 

    Sometimes you are working on a project with a version being used by many people and you may not want to edit that version.

So you create a branch with the command
git checkout -b branchname

* To see what branch you are on type:
git branch

* To switch back to the master branch type
git checkout master

    Pull Request 

    If you fork someone's repo or have multiple branches you will be working seperately
Sometimes you want to merge in your changes into the other branc/repo
To do so you need to send a pull request
